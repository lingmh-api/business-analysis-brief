# 一页立项简报：AI驱动的企业经营分析简报自动生成系统

## 一、业务痛点（问题定义）

企业月度/季度经营分析会上，财务与业务团队面临三大痛点：
1. **数据分散、整合耗时**：经营数据分散在ERP、CRM、财务系统等多个平台，人工汇总耗时2-3天；
2. **分析依赖经验，容易遗漏**：KPI异常检测靠人工经验，容易遗漏跨维度关联异常；
3. **简报撰写重复低效**：管理层需要的经营简报格式固定但撰写耗时，分析师60%时间花在"排版填数"而非"洞察分析"。

## 二、AI解决路径

```
原始经营数据 → [AI异常检测] → 异常指标定位 → [AI归因分析] → 自然语言简报生成 → 管理层简报
                                  ↑
                           NL2SQL自然语言问数（管理层可自助追问）
```

1. **AI异常检测**：基于历史数据训练/提示，自动识别多维度KPI异常波动
2. **NL2SQL问数**：管理层可用自然语言提问（如"华南区上月毛利率为什么下降？"），AI自动转SQL查询
3. **简报自动生成**：基于分析结论，调用大模型自动生成结构化经营简报（含文字分析+图表解读）

## 三、所需数据

| 数据类别 | 具体指标 | 来源 |
|----------|----------|------|
| 收入数据 | 分区域/产品线/渠道的销售收入、同比环比 | ERP/财务系统 |
| 成本数据 | 毛利率、费用率、成本构成 | 财务系统 |
| 运营数据 | 库存周转、应收账款周转、交付及时率 | ERP/SCM |
| KPI基准 | 历史同期数据、预算目标值 | 预算系统 |

## 四、AI工具选型

- **大模型**：DeepSeek API / 通义千问API（用于简报生成、NL2SQL、归因分析）
- **数据分析**：Python Pandas + NumPy（数据清洗与统计）
- **可视化**：Matplotlib + Plotly（图表生成）
- **前端展示**：Streamlit（轻量级交互界面）

## 五、预期业务价值

| 维度 | 传统方式 | AI赋能后 | 提升 |
|------|----------|----------|------|
| 数据整合 | 2-3天人工汇总 | 分钟级自动拉取 | 效率↑95% |
| KPI异常发现 | 人工经验，遗漏率~30% | AI全量扫描，遗漏率<5% | 准确率↑25% |
| 简报生成 | 1-2天撰写排版 | 分钟级自动生成 | 效率↑99% |
| 问数能力 | 需技术人员取数 | 自然语言自助查询 | 门槛↓100% |

## 六、答辩三问预答

**① 解决什么问题？**
解决企业经营分析中"数据整合慢→异常发现漏→简报撰写累"的链式痛点，让分析师从"表哥表姐"转型为"业务洞察者"。

**② AI扮演什么角色？**
AI是核心引擎——异常检测替代人工扫表、NL2SQL替代技术人员取数、大模型生成替代手动撰写。非AI不可，因为规则引擎无法理解语义、无法生成自然语言洞察。

**③ 业务价值？**
将经营简报生产周期从3-5天压缩到分钟级，异常发现准确率提升25%，管理层可自助用自然语言探索数据。
